# 人物分析框架

长篇网文的人物设计分析方法。包含人物元素提取、关系映射、动机链分析和跨章节一致性检查。

---

## 一、人物元素提取

### 提取模板

```
角色名：
性别：
角色定位：主角 / 对手 / 盟友 / 导师 / 工具人 / 隐藏角色
身份标签：{社会身份，如「落魄世家子弟」「金牌律师」「村霸」}
外貌关键词：{3-5 个词，不是详细描写，是关键特征}
性格关键词：{3-5 个核心性格词}
核心目标：{这个角色最想达成什么}
深层动机：{为什么想达成这个目标（和目标不同）}
致命弱点：{这个角色的软肋/盲点}
口头禅/标志性动作：{}
```

### 提取要点

- **身份标签**比详细背景更重要。标签决定读者对这个角色的第一判断
- **核心目标**和**深层动机**要分开看。目标是他做什么，动机是他为什么做。好人物的目标和动机常有张力（表面追求 A，实际渴望 B）
- **致命弱点**是推动剧情的关键。没有弱点的角色是工具人，有弱点的角色才是人

### 角色功能分类

| 类型 | 功能 | 判断标准 |
|------|------|----------|
| 主角 | 读者代入 + 推动主线 | 视角最多、成长弧线最完整 |
| 对手 | 制造冲突 + 推动成长 | 和主角目标直接对立 |
| 盟友 | 辅助主角 + 提供情感价值 | 在关键时刻帮助主角 |
| 导师 | 传授能力 + 指引方向 | 比主角强、阶段性出场 |
| 工具人 | 推动某个情节 | 功能单一、出场后可退场 |
| 隐藏角色 | 制造反转/悬念 | 前期不露面但影响全局 |

## 二、人物关系映射

### 四种关系类型

#### 1. 冲突关系

```
角色 A ←→ 角色 B
冲突类型：利益冲突 / 价值观冲突 / 情感冲突 / 生存冲突
冲突强度：1-10
触发事件：{什么引起了冲突}
升级方式：{冲突如何一步步加剧}
```

分析要点：
- 冲突是否有升级路线（小摩擦 → 大对抗 → 生死战）
- 冲突的根源是否足够深（不是单纯的脸谱化对立）
- 冲突是否能推动双方的角色发展

#### 2. 结盟关系

```
角色 A ←→ 角色 B
结盟基础：利益一致 / 情感羁绊 / 共同敌人
信任程度：低 / 中 / 高 / 绝对信任
潜在裂痕：{什么可能导致联盟破裂}
```

分析要点：
- 结盟是否有利益基础（不是无缘无故的帮忙）
- 是否有潜在矛盾（盟友间可以有分歧，增加真实感）
- 结盟是否推动主线发展

#### 3. 亲密关系

```
角色 A ←→ 角色 B
关系类型：恋人 / 家人 / 师徒 / 知己
情感深度：表面 / 深层 / 灵魂级
关键场景：{定义这段关系的核心场景}
变化轨迹：{从陌生到亲密的过程，或从亲密到决裂}
```

分析要点：
- 亲密关系的建立是否有足够的铺垫和事件支撑
- 关系变化是否推动剧情（不只是情感线）
- 是否有独特的互动细节让这段关系区别于其他作品

#### 4. 权力关系

```
角色 A（上位）→ 角色 B（下位）
权力来源：身份 / 实力 / 资源 / 信息
权力动态：固定 / 翻转中 / 已翻转
```

分析要点：
- 权力关系是否有动态变化（权力翻转是大爽点来源）
- 权力差是否被合理利用（制造压迫感/释放爽感）
- 翻转是否有铺垫（不能无理由翻转）

### 关系图绘制

对全书或一卷绘制人物关系网：

```
中心：主角
第一圈：与主角直接互动的角色（标注关系类型和强度）
第二圈：通过第一圈角色间接影响主角的角色
标注：权力方向（→）、冲突强度（粗线）、亲密程度（虚线）
```

## 三、动机链分析

### 动机链模型

```
起因（Cause）
  → 触发事件：{什么事件触发了角色的行动}
  → 角色处境：{事件发生时角色的状态}

意图（Intent）
  → 表面意图：{角色说自己要做什么}
  → 真实意图：{角色实际想达到什么}
  → 意图层级：{表层目标 → 中层需求 → 深层渴望}

约束（Constraint）
  → 外部约束：{客观条件限制（实力不够/没有资源/被人阻止）}
  → 内部约束：{角色自身的限制（性格弱点/道德底线/情感羁绊）}

风险（Risk）
  → 失败代价：{如果行动失败会失去什么}
  → 成功代价：{如果行动成功要牺牲什么}
  → 道德代价：{这个行动是否有道德争议}
```

### 动机链检查清单

- [ ] 每个重要行动是否有明确的起因（不是无缘无故）
- [ ] 表面意图和真实意图是否有区分（复杂的角色不会直说真实想法）
- [ ] 约束是否合理（不能让角色太容易达成目标）
- [ ] 风险是否足够（高风险才有关注度）
- [ ] 角色的关键抉择是否反映动机链（在约束和风险下的选择最能体现角色本质）

### 动机冲突类型

| 类型 | 定义 | 经典例子 |
|------|------|----------|
| 目标 vs 道德 | 想做的事和应该做的事冲突 | 复仇但不想伤及无辜 |
| 情感 vs 利益 | 感性和理性的拉扯 | 知道对方不可靠但放不下 |
| 短期 vs 长期 | 眼前利益和远大目标的矛盾 | 现在妥协保命 vs 坚持原则赌未来 |
| 自我 vs 他人 | 个人需求和他人期望的冲突 | 想过平静生活但背负家族使命 |

## 四、跨章节一致性检查

### 检查维度

#### 1. 性格一致性

```
检查项：角色在不同场景下的行为是否符合同一性格设定
方法：
  - 列出角色的核心性格词
  - 在 5 个关键场景中标记角色的行为
  - 检查行为是否和性格词一致
注意：
  - 角色可以成长和变化，但变化要有铺垫
  - 压力下的行为最能体现真实性格
  - 「在 XX 情况下会做 YY」是比性格词更精准的描述
```

#### 2. 关系一致性

```
检查项：角色间的关系变化是否有迹可循
方法：
  - 按章节顺序列出 A-B 关系的关键变化点
  - 检查每个变化点是否有触发事件
  - 检查是否有突然变化但缺乏铺垫的情况
常见问题：
  - 关系突然变好/变差没有理由
  - 前面的矛盾后面突然不存在了
  - 亲密关系缺少建立过程
```

#### 3. 能力一致性

```
检查项：角色的实力/能力是否前后一致
方法：
  - 标记角色每次展示实力的场景
  - 对比同类型场景下角色的表现是否一致
常见问题：
  - 主角有时强有时弱，取决于剧情需要（战力崩坏）
  - 配角在有主角在场时莫名变弱
  - 设定的限制在后面被遗忘
```

#### 4. 信息一致性

```
检查项：角色知道什么、不知道什么是否前后一致
方法：
  - 列出关键信息的揭露时间点
  - 检查角色在某个时间点之前不应该知道某些信息
  - 检查角色是否利用了不应该知道的信息
常见问题：
  - 重生文中最常见：主角使用了前世记忆中没有的信息
  - 角色在 A 章不知道的事在 B 章突然知道了但没人告诉他
  - 悬疑文中：角色的行为暗示他知道真相但设定上他还不知道
```

### 一致性检查频率建议

| 检查时机 | 检查范围 | 重点关注 |
|----------|----------|----------|
| 每 10-20 章 | 最近章节的所有角色 | 性格 + 关系 |
| 每卷结束 | 本卷所有角色 + 跨卷主线角色 | 能力 + 信息 |
| 全书完稿 | 所有重要角色 | 全部四维度 |

### 检查输出格式

```
角色：{名字}
检查范围：第 X 章 ~ 第 Y 章

问题 1：{问题描述}
  - 位置：第 N 章
  - 前文设定：{之前怎么设定的}
  - 当前矛盾：{现在怎么写的}
  - 严重程度：高/中/低
  - 修复建议：{怎么改}

问题 2：...
```
