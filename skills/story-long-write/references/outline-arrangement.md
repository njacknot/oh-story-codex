# 大纲排布方法论

大纲的定义：在故事核的基础上，按故事架构的指引排布剧情，敲定故事线上的剧情和需预埋的信息点。

## 故事结构分级

| 级别 | 特征 | 适合 |
|------|------|------|
| 一级 | 单元剧松散连接，靠时间串联，最后大高潮 | 入门作者 |
| 1.5 级 | 单元剧之间有逻辑线索串联，指向核心目标 | 进阶作者 |
| 二级 | 小高潮 → 意外下行 → 上行转折 → 大高潮链 | 高阶作者 |

## 五步大纲创建法

### 第一步：确定高潮剧情

高潮有绝对优先级——最大冲突规模、最多出场人物、最强情绪波动。

检查清单：
- [ ] 这个场景是否是本卷最大的冲突？
- [ ] 出场人物是否最多？
- [ ] 是否与阶段目标高度一致？
- [ ] 读者情绪波动是否最大？

### 第二步：确定单元剧

单元剧 = 主角如何使用金手指优势获得爽点。

同类型的故事重复出现 = **剧情套路**
有逻辑关系的故事类型成组出现 = **剧情套路链路**

设计要点：
- 每个单元剧都要展示金手指的不同用法
- 相邻单元剧不能使用相同的金手指逻辑
- 单元剧之间要有因果关系，不能纯平行

### 第三步：故事线组织与信息点预埋

需要同时管理 8 条故事线：

| 故事线 | 内容 | 预埋什么 |
|--------|------|----------|
| 地图线 | 层级地点（城→地标→室内） | 新地图解锁时机 |
| 阵营线 | 势力与地图匹配 | 阵营冲突爆发点 |
| 人物线 | 角色按类型分组 | 新角色登场时机 |
| 金手指线 | 主角获得的物品/技能 | 能力升级节点 |
| 世界观线 | 世界设定规则 | 设定揭示时机 |
| 矛盾线 | 冲突放入矛盾网络 | 矛盾升级链 |
| 收集线 | 材料/线索收集 | 3+ 渐进式收集层级 |
| 感情线 | 感情关系进展 | 亲密度升级节点 |

### 第四步：确定开篇舞台

开篇模板：
```
情节钩子 → 迅速陷入异常状态 → 获得/介绍金手指 → 矛盾/欲望出现，主角运转金手指 → 目标建立
```

必须在开篇预埋的期待感：
- 新目标/冲突升级的方向
- 未来实力升级和新技能的暗示
- 未来资源升级的暗示
- 未来感情线升级的暗示
- 未来人际网络扩展的暗示
- 混乱程度升级的暗示
- 新线索/新设定的暗示

### 第五步：确定收尾

收尾功能清单：
- [ ] 解决主线/副线高潮，展示后续影响
- [ ] 主角获得足够奖励 + 额外奖励（用于下一阶段）
- [ ] 给本卷角色合适的结局
- [ ] 节奏放缓
- [ ] 为下一卷内容做铺垫

## 剧情质量控制

### 剧情相似度检查

比较两个剧情在以下维度是否过于相似：
- 冲突/矛盾类型
- 金手指使用方式
- 剧情链路段落
- 剧情结局

### 剧情匹配度检查

| 维度 | 检查内容 |
|------|----------|
| 阶段匹配 | 剧情/感情线进度是否匹配本卷阶段 |
| 场景匹配 | 场景是否在本卷场景列表内 |
| 人物匹配 | 出场人物组合、关系、行为是否一致 |

### 约束规则

- 时空关联 : 逻辑关联 ≥ 1:3（每 1 个因果推进，至少 3 个场景推进）
- 后续剧情情绪值不能显著低于前序
- 相同金手指逻辑不连续使用
- 同类剧情套路间隔 ≥ 3 个不同类型剧情
